Quick Start

Get up and running with Aegis in under 5 minutes. This guide will walk you through installation, authentication, and your first API call.

Prerequisites: You'll need an API key. Contact sales to get started.

1. Install the SDK

# Python
pip install aegis-shield

# Node.js
npm install @aegis-shield/sdk

# Go
go get github.com/aegis-shield/aegis-go

2. Initialize the Client

from aegis_shield import AegisClient

# Initialize with your API key
client = AegisClient(
    api_key="sk_live_YOUR_API_KEY",
    environment="production"  # or "sandbox" for testing
)

3. Process Your First Message

# User message containing PII
user_message = "My name is John Smith, SSN 123-45-6789"

# Process with Aegis before sending to LLM
result = client.process(
    text=user_message,
    method="auto"  # Automatically selects best method
)

# Send clean text to OpenAI/Claude
clean_text = result.processed_data
# Output: "My name is [PERSON_NAME], SSN [SSN_REDACTED]"

Direct API Access

If we don't have an SDK for your language yet, you can use the REST API directly:

curl -X POST https://api.aegis-shield.ai/v1/process \
  -H "Authorization: Bearer sk_live_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": "Text containing PII",
    "method": "auto"
  }'

Authentication

All API requests require authentication using an API key in the Authorization header.

API Key Format

Your API key will look like this:

sk_live_abcdef123456789...

Environment Variables

We recommend storing your API key in environment variables:

# .env file
AEGIS_API_KEY=sk_live_YOUR_API_KEY

# Python
import os
client = AegisClient(api_key=os.getenv("AEGIS_API_KEY"))

# Node.js
const client = new AegisClient({
    apiKey: process.env.AEGIS_API_KEY
})
Security: Never commit API keys to version control. Use environment variables or secret management systems.

Privacy Methods

Aegis offers six different privacy preservation methods. Choose based on your use case:

Redaction

Complete removal of PII. Best for maximum security.

Input: "SSN 123-45-6789" Output: "SSN [REDACTED]"

Masking

Partial hiding. Useful for verification while maintaining privacy.

Input: "Card 4532-1234-5678-9012" Output: "Card ****-****-****-9012"

Tokenization

Reversible replacement. Perfect for medical records.

Input: "Patient John Smith" Output: "Patient TOKEN_847362"

Differential Privacy

Statistical noise for analytics. Preserves trends, not individuals.

Input: "Age 45, Salary $95,000" Output: "Age 40-50, Salary $90-100k"

K-Anonymity

Generalization for group privacy. FERPA compliant.

Input: "123 Main St, Apt 4B" Output: "Downtown District"

Auto (Recommended)

AI selects optimal method based on context and compliance.

Automatic selection based on: • Data type • Industry • Regulations

OpenAI GPT-4 Integration

Protect user data before sending to OpenAI's API:

from openai import OpenAI
from aegis_shield import AegisClient

# Initialize both clients
aegis = AegisClient(api_key="sk_live_YOUR_AEGIS_KEY")
openai = OpenAI(api_key="YOUR_OPENAI_KEY")

def safe_chat_completion(user_message):
    # Step 1: Clean with Aegis
    protection = aegis.process(
        text=user_message,
        method="auto"
    )

    # Step 2: Send to OpenAI
    response = openai.chat.completions.create(
        model="gpt-4",
        messages=[
            {"role": "system", "content": "You are a helpful assistant."},
            {"role": "user", "content": protection.processed_data}
        ]
    )

    # Step 3: Check response for generated PII
    clean_response = aegis.process(
        text=response.choices[0].message.content,
        method="detection"  # Only detect, don't modify
    )

    return {
        "response": clean_response.processed_data,
        "pii_removed": protection.entities_detected,
        "compliance": protection.compliance_status
    }

# Usage
result = safe_chat_completion("My SSN is 123-45-6789, can you help?")
print(result["response"])  # Safe response, no PII
